Description

EmergeNav is an emergency assistance device designed to help users quickly locate emergency services. The device integrates several components and services, including a TI CC3200 microcontroller with Wi-Fi, a GPS module, AWS IoT services, AWS Lambda Services, Google Places & Directions API, an Adafruit OLED display, and AWS SNS for notifications.

Design

Functional Specification

The Finite State Machine (FSM) for this emergency response system manages transitions between various states to handle emergencies. Starting in an idle state, the system activates upon user interaction by pressing an emergency button on the TI CC3200 board. It then displays service options on the Adafruit OLED module and, based on the user's selection, coordinates with AWS IoT services to dispatch GPS coordinates and service preferences. Utilizing the Google Maps API, the system identifies the nearest emergency facilities and sends precise directions to the TI CC3200 board and the user's email via AWS SNS. After ensuring the necessary responses are in place, the FSM returns to its idle state, prepared to handle future emergencies.

System Architecture

Flow Diagram and Components

The major components of this system are TI CC3200 microcontroller, IR Remote receiver, the Adafruit SSD1351 OLED, NEO-6M GPS Module and AWS IOT Web Services.

  • TI CC3200 Microcontroller

    The TI CC3200 microcontroller is a versatile chip designed for IoT applications, integrating a powerful ARM Cortex-M4 MCU and Wi-Fi connectivity. The CC3200 includes multiple GPIO pins for interfacing with various devices, UART for serial communication, and SPI for high-speed data transfer.

  • IR Receiver

    Modulation in the context of infrared (IR) communication involves varying a carrier wave to encode information that can be transmitted through space as IR light signals. Typically, a base frequency for the carrier wave is chosen, which aligns with the optimal sensitivity of the IR receiver. For many consumer electronics, such as remote controls, a common frequency is around 36 kHz.

  • Adafruit SSD1351 OLED

    The OLED SSD1351 128 RGB display module connects to a microcontroller via several key pins: VCC (power), GND (ground), SCL/SCK (serial clock for SPI), SDA/MOSI (serial data for SPI), DC (data/command control), CS (chip select to enable/disable the module), and RST (reset). Proper matching of these pins with the microcontroller's SPI interface is crucial for the display's operation.

  • NEO-6M and Patch Antenna

    The NEO-6M module contains a built-in receiver that picks up signals transmitted at the speed of light by GPS satellites orbiting the Earth. Each GPS satellite transmits data that includes its location and the exact time the signal was sent. Using the method of trilateration, the NEO-6M module can determine its precise location. Additionally, GPS satellites contain atomic clocks that provide precise timing. Once the receiver calculates its position, it uses National Marine Electronics Association (NMEA) format to output its data. This output typically includes information like current latitude, longitude, time, altitude, and almanac data.

    The patch antenna used with the NEO-6M GPS module is a critical component, specifically tuned to the L1 frequency band of 1575.42 MHz to optimally receive GPS signals. Patch antennas predominantly radiate off the edges of the patch. Due to their flat design, patch antennas emit radio waves primarily upwards when mounted horizontally. Additionally, patch antennas can be designed to support linear or circular polarization to receive data from GPS.

  • AWS IoT core

    AWS IoT Core provides cloud management for devices. Device Shadow ensures consistent device state representation. It allows devices to update their state even when offline. This enables seamless device-cloud interaction. For example, devices can update their shadow state, reflecting changes in the cloud, facilitating robust IoT solutions.

  • AWS Lambda

    AWS Lambda is a serverless computing service that implements a code in response to events without provisioning or managing servers. This seamless integration exemplifies the power and flexibility of AWS services in building robust IoT applications

  • AWS SNS

    Amazon Simple Notification Service (SNS) is a fully managed messaging service offered by Amazon Web Services (AWS). It enables the delivery of messages to a variety of endpoints, including email, SMS, HTTP, and more. With SNS, you can easily send notifications, alerts, and messages to distributed systems and applications, ensuring timely communication and efficient handling of events. SNS provides a reliable and scalable solution for delivering notifications, enhancing the overall functionality and user experience of your applications.

Implementation

The IR receiver connects power (Vs), ground (GND), and output (OUT) pins, with a resistor and capacitor filtering noise, and outputs to pin 64, configured via SysConfig. The Neo-6M GPS uses UART to send NMEA format GPS data to the CC3200, with pins 58 and 59 set for UART1. The Adafruit OLED connects to the CC3200: VCC , GND, DC (pin 15), CS (pin 3), SCK (pin 5), Reset (pin 4), and MOSI (pin 7), enabling real-time data processing and display.

AT&T Remote and IR sensor

The TV remote that we used, used an NEC communication protocol The IR Receiver is configured to capture and decode signals from a remote control. We connected the IR receiver's OUT signal to an unused GPIO pin, configuring it as an input using the SysConfig tool. Our system used interrupts to detect edges of the IR signal and used the microcontroller’s SysTick module to measure pulse widths. The pin that was used to read the output of the IR Receiver was pin64.

GPS Module and UART

The Neo-6M GPS module uses UART (Universal Asynchronous Receiver/Transmitter) communication to interface with the CC3200 board. Through its TX (transmit) and RX (receive) pins, the Neo-6M sends GPS data, including latitude, longitude, altitude, and time, in NMEA (National Marine Electronics Association) format. The UART communication ensures that the GPS data is accurately transmitted at a specified baud rate, typically 9600 bps. Using the SysConfig tool, we configured pins 58 and 59 for UART1 as transmitter and receiver pins, which were then connected to the GPS module.

OLED and SPI

The Adafruit SSD1351 OLED module interfaces with the CC3200 board through the following pin connections: VCC (3.3V-5V) for power supply, GND for ground, DC (connected to pin 15) for differentiating command/data signals, CS (connected to pin 3) for selecting the OLED display, SCK (connected to pin 5) for clock signal synchronization, Reset (connected to pin 4) for resetting the device, and MOSI (connected to pin 7) for data transmission from the master device.

AWS IoT core, Lambda and SNS

Our IoT solution utilizes AWS IoT Core to manage device-cloud interactions, employing device shadows to persistently represent device states. A POST function updates the device shadow with coordinate data and emergency service types through AWS IoT Core's REST API endpoint. This mechanism ensures the device state reflects the latest information, facilitating seamless communication between devices and cloud applications. Additionally, AWS Lambda functions are triggered by incoming device data, processing it, making API calls to Google Directions & Places for nearby emergency services, and updating the device shadow accordingly. The Lambda function automates this workflow, enhancing the IoT solution by dynamically enriching device data with contextually relevant information from external APIs. By integrating SNS, users receive timely alerts, ensuring they're informed in critical situations.This capability enhances user experience and underscores the versatility of AWS services in building comprehensive IoT solutions.

Challenges

The project faced several challenges, including integrating a GPS module that only functioned outdoors, complicating system integration. Connecting to AWS through Wi-Fi and ensuring GPS functionality indoors posed additional difficulties. Realizing the limitations of the Google Maps API, the team switched to the Google Directions and Places API for better nearest location information. However, adapting to the new API involved parsing JSON output, which became problematic when the API's format unexpectedly changed overnight. Overcoming these challenges demanded technical expertise, flexibility, and problem-solving skills from the project team.

Future Work

Future work includes enhancing the device's capabilities by increasing its accuracy, range, and integrating additional sensors. We aim to improve the hardware with a GPS module that functions both indoors and outdoors. Another key development area is the user interface, focusing on making it more intuitive and user-friendly, including a graphical user interface (GUI) to guide users with step-by-step instructions and troubleshooting tips. Additionally, we plan to expand service integration by incorporating more third-party APIs and services, thereby improving the device's overall utility. These enhancements will lead to a more robust, versatile, and user-centric solution.
